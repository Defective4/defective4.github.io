Controlling a LED segment display with a Raspberry Pi




Segment displays usually require special circuits to control. Of course, you can manually light up each of the display's segments, but it's hard to display any useful data that way.
But what if we could control each segment using a computer?

Premise

We'll use a Raspberry Pi's GPIO pins, which can conveniently output 3.3V and can be controlled however we want!
For displays with a common cathode we can just wire each segment to its own GPIO pin, of course with resistors in series.

The common anode problem

There's a catch though. If you only have a common anode display, then you are in quite a situation.
In common anode displays each segment is controlled by negative voltage, and the common pin is positive. That's a problem, because GPIO pins can only output positive voltage!
My solution was to use one NPN transistor per segment. Each transistor's emitter can easily be grounded, and the collector wired through a resistor with a led segment. And then we can control the transistor by wiring its base through a resistor to a GPIO pin.
It's not the prettiest solution, but it sure is easiest!
So, let's remake our schematic for a common anode display:



This approach is actually way safer than the first one!
Each transistor requires less than 1 mA of base current to saturate, guaranting that our Raspberry will never be overloaded.
Raspberry's 3.3V rail (including GPIO pins) can only output less than 50 mA simultaneously, and driving each LED segment directly from GPIO can take more than half of that limit!

Testing

After assembling our circuit it's time to test it!
To do so, we can use the raspi-gpio command included with the Raspberry OS.
Assuming you are using tbe same pins as in schematics above (17, 18, 22, 23, 24, 25, 27), we first need to set all ports' function to output.
We can do that with a simple bash command:

for pin in 17 18 22 23 24 25 27; do raspi-gpio set $pin op; done

Now we can test if all pins are wired correctly, by using

raspi-gpio set <pin number> dh

and then

raspi-gpio set <pin number> dl

If respective pins light up, then everything works!
Now on to the next part...

Programming

I will use a Java library called pi4j to interact with Raspberry's outputs.
You can also skip this part and see my repository dedicated to this project.

This section assumes you know how to setup a Java Maven projects with pi4j library. Of course basic Java knowledge is also required.
After setting up a project, we will create a class, that will be our interface to the display:

import com.pi4j.context.Context;
import com.pi4j.io.gpio.digital.DigitalOutput;

public class DisplayController {
    private final DigitalOutput[] leds;

    public DisplayController(Context pi4j, int[] pins) {
        if (pins.length != 7) throw new IllegalArgumentException("pins != 7");
        leds = new DigitalOutput[pins.length];
        for (int x = 0; x < pins.length; x++) {
            leds[x] = pi4j.digitalOutput().create(pins[x]);
        }
    }
}

As you can see, it takes a pi4j Context instance as its argument, and a int array.
The passed integer array should contain addresses of all our 7 GPIO pins, each corresponding to a display segment.
For the sake of this post, let's assume the order of pins is as follows: A, F, B, G, E, C, D.
See schematics above for reference.
Now we need to map all possible digits to segments they require to be fully displayed.
We will create a DisplayCharacter enum:

public enum DisplayCharacter {
    ;

    private final int[] segments;
    private final char character;

    private DisplayCharacter(char c, int... segments) {
        character = c;
        this.segments = segments;
    }

    public int[] getSegments() {
        return segments;
    }

}

The enum will hold all characters along with their segment indexes.

The ONE digit needs segments 2 and 5 to light, TWO needs 0, 2, 3, 4, 6, and so on...
Now we need some convenience methods:

    private DisplayCharacter(char c, int... segments) {
        character = c;
        this.segments = segments;
    }

    public int[] getSegments() {
        return segments;
    }

    public static DisplayCharacter getForDigit(int i) {
        if (i < 0 || i > 9) throw new IllegalArgumentException("Only single digits are supported");
        return getForCharacter((char) (i + 48));
    }

    public static DisplayCharacter getForCharacter(char c) {
        for (DisplayCharacter dc : values()) if (dc.c2 == c) return dc;
        return null;
    }

    public static boolean isValidCharacter(char c) {
        return getForCharacter(c) != null;
    }

}

...and our enum is ready to use!
Back to our DisplayController, we now need to add some methods to actually display the characters.
First we need a clear method, to turn off all outputs and cause the display to go blank (if it isn't already). It will also help us change displayed characters, clearing the display between changes.

import com.pi4j.context.Context;
import com.pi4j.io.gpio.digital.DigitalOutput;

public class DisplayController {
    private final DigitalOutput[] leds;

    public DisplayController(Context pi4j, int[] pins) {
        if (pins.length != 7) throw new IllegalArgumentException("pins != 7");
        leds = new DigitalOutput[pins.length];
        for (int x = 0; x < pins.length; x++) {
            leds[x] = pi4j.digitalOutput().create(pins[x]);
        }
    }

    public void clear() {
        for (DigitalOutput led : leds) led.setState(false);
    }
}

Here is our controller class with clear method added.
It loops through all available outputs, and sets them all to LOW state.
Our next method, displayCharacter, is quite similar.
It will loop through all outputs, and light the ones, that match with the provided DisplayCharacter!

import com.pi4j.context.Context;
import com.pi4j.io.gpio.digital.DigitalOutput;

public class DisplayController {
    private final DigitalOutput[] leds;

    public DisplayController(Context pi4j, int[] pins) {
        if (pins.length != 7) throw new IllegalArgumentException("pins != 7");
        leds = new DigitalOutput[pins.length];
        for (int x = 0; x < pins.length; x++) {
            leds[x] = pi4j.digitalOutput().create(pins[x]);
        }
    }

    public void clear() {
        for (DigitalOutput led : leds) led.setState(false);
    }
    
    public void displayCharacter(DisplayCharacter character) {
        clear();
        for (int l : character.getSegments()) leds[l].setState(true);
    }
}

And here is our complete class.
Now to use it:

import com.pi4j.Pi4J;
import com.pi4j.context.Context;

public class DisplayMain {

    public static void main(String[] args) {
        Context pi4j = Pi4J.newAutoContext();
        int[] pins = {
                18, 27, 25, 24, 23, 22, 17
        };
        DisplayController display = new DisplayController(pi4j, pins);
        display.displayCharacter(DisplayCharacter.getForDigit(4));
    }

}

After compiling and exporting the code, you can run it.
Remember that you need to use sudo, otherwise the library won't be able to use GPIO!
If all is done correctly, you will now see a 4 displayed on your display.
